Topic 1 of 8 — the absolute start
Variables, types & mutability

What Python stores in a variable

foundation for everything
A variable is just a label pointing to an object in memory. Python doesn't care about the type when you declare — the object itself knows its type. This matters for DSA because knowing whether something is mutable (can change in place) or immutable (cannot change) determines how copying and function arguments behave.
Core types — know all of these
# Immutable — cannot be changed after creation x = 42 # int pi = 3.14 # float name = "arif" # str flag = True # bool t = (1, 2, 3) # tuple — immutable list # Mutable — can change in place (critical for DSA) arr = [1, 2, 3] # list d = {"a": 1} # dict s = {1, 2, 3} # set
Mental model: Think of a variable as a sticky note on a box. Multiple sticky notes can point to the same box. With mutable objects, editing the box changes it for everyone pointing at it.
The mutation trap — extremely common DSA bug
a = [1, 2, 3] b = a # b points to the SAME list b.append(4) print(a) # [1, 2, 3, 4] ← a changed too! # Fix: make a real copy b = a.copy() # or a[:] or list(a) b.append(4) print(a) # [1, 2, 3] ← a unchanged
Type checking & conversion
type(42) # <class 'int'> isinstance(42, int) # True int("123") # 123 str(42) # "42" float("3.14") # 3.14 list((1,2,3)) # [1, 2, 3] — tuple to list
DSA connection: When building DP memoization tables or returning lists from recursion, you MUST use copy() or list(arr) when saving state — otherwise all saved states point to the same mutating list.

Topic 2 of 8
Control flow — loops, conditionals, break/continue

For loops — the DSA workhorse

used in every single algorithm
Every loop form you need for DSA
# Basic index loop for i in range(5): # 0 1 2 3 4 pass # Loop with index AND value — use this often arr = [10, 20, 30] for i, val in enumerate(arr): print(i, val) # 0 10 / 1 20 / 2 30 # Reverse loop — common in DSA problems for i in range(len(arr)-1, -1, -1): print(arr[i]) # 30 20 10 # Zip — loop two lists in parallel a = [1,2,3] b = [4,5,6] for x, y in zip(a, b): print(x+y) # 5 7 9
While loop — essential for two-pointer, binary search
# Two-pointer skeleton — you'll write this constantly left, right = 0, len(arr) - 1 while left < right: # do something with arr[left] and arr[right] left += 1 right -= 1 # Binary search skeleton lo, hi = 0, len(arr) - 1 while lo <= hi: mid = (lo + hi) // 2 if arr[mid] == target: return mid elif arr[mid] < target: lo = mid + 1 else: hi = mid - 1
Break and continue — use deliberately
# break — exit loop entirely (e.g., found target) for i, v in enumerate(arr): if v == target: result = i break # stop searching # continue — skip this iteration for x in arr: if x % 2 == 0: continue # skip evens print(x) # only odd numbers # else on a loop — fires if no break occurred for x in arr: if x == target: break else: print("not found") # only if loop didn't break
Common bug: Modifying a list while looping over it. Never do for x in arr: arr.remove(x) — the indices shift under you. Loop over a copy: for x in arr[:]: or collect indices to remove and delete after.

Topic 3 of 8
Lists — indexing, slicing, methods

Indexing & slicing — your most-used tool

in every array problem
All indexing forms
arr = [10, 20, 30, 40, 50] # 0 1 2 3 4 ← forward index # -5 -4 -3 -2 -1 ← backward index arr[0] # 10 — first element arr[-1] # 50 — last element (critical for DSA) arr[-2] # 40 — second to last # Slicing: arr[start:stop:step] (stop is EXCLUSIVE) arr[1:4] # [20, 30, 40] arr[:3] # [10, 20, 30] — from start arr[2:] # [30, 40, 50] — to end arr[::-1] # [50, 40, 30, 20, 10] — reversed! arr[1:4:2] # [20, 40] — every 2nd
Critical list methods — complexity matters
# O(1) — fast — use freely arr.append(x) # add to end arr.pop() # remove from end arr[-1] # peek last element len(arr) # length # O(n) — slow — use carefully in nested loops arr.insert(0, x) # O(n) — shifts everything right arr.pop(0) # O(n) — shifts everything left x in arr # O(n) — linear scan arr.remove(x) # O(n) — finds then removes arr.index(x) # O(n) — finds first occurrence # Sorting arr.sort() # in-place, O(n log n) arr.sort(reverse=True) sorted(arr) # returns new list, original unchanged arr.sort(key=lambda x: x[1]) # sort by index 1
List comprehensions — write these fluently
# Basic: [expression for item in iterable] squares = [x**2 for x in range(5)] # [0, 1, 4, 9, 16] # With filter: [...if condition] evens = [x for x in range(10) if x % 2 == 0] # [0, 2, 4, 6, 8] # 2D — creating grids (for matrix problems) grid = [[0] * 3 for _ in range(3)] # [[0,0,0],[0,0,0],[0,0,0]] # WRONG way — all rows share same list! bad = [[0] * 3] * 3 # don't do this
DSA connection: The 2D grid creation pattern is used constantly for DP tables, visited matrices in graph problems, and memoization tables. Always use the list comprehension form, never the * n shortcut.

Topic 4 of 8
Dicts & sets — O(1) lookup power

Dictionaries — the most important DS for DSA

reduces O(n²) → O(n) constantly
Dicts give you O(1) average lookup, insert, and delete. Any time you're doing a nested loop to check if something exists, a dict can make it a single pass. This is the single most valuable optimization skill in DSA interviews.
All dict operations
d = {"a": 1, "b": 2} # Access d["a"] # 1 d.get("c", 0) # 0 — safe: returns default if missing # Insert / update d["c"] = 3 # Check existence — O(1) "a" in d # True # Iterate for k, v in d.items(): print(k, v) for k in d.keys(): print(k) for v in d.values(): print(v) # Delete del d["a"] d.pop("b", None) # safe pop
defaultdict & Counter — the DSA essentials from collections
from collections import defaultdict, Counter # defaultdict — no KeyError, auto-creates default value freq = defaultdict(int) # default 0 for char in "hello": freq[char] += 1 # {'h':1, 'e':1, 'l':2, 'o':1} graph = defaultdict(list) # default [] graph[0].append(1) # adjacency list # Counter — counts elements instantly c = Counter("hello") # Counter({'l': 2, 'h': 1, 'e': 1, 'o': 1}) c.most_common(2) # [('l', 2), ('h', 1)] # Two-sum pattern — the canonical O(n) solution def two_sum(nums, target): seen = {} for i, n in enumerate(nums): complement = target - n if complement in seen: return [seen[complement], i] seen[n] = i
Mental model: A dict is like a cloakroom. You give a key (coat ticket), you get back your value instantly — the attendant doesn't search the whole room. This is why it's O(1) and a list scan is O(n).

Sets — instant membership testing

visited tracking in graph/BFS/DFS
Set operations
s = {1, 2, 3} s.add(4) # O(1) s.discard(2) # O(1) — no error if missing 3 in s # True — O(1) 5 in s # False — O(1) # Convert list → set removes duplicates list(set([1,2,2,3])) # [1, 2, 3] (order not guaranteed) # Set math a = {1,2,3}; b = {2,3,4} a & b # {2, 3} — intersection a | b # {1,2,3,4} — union a - b # {1} — difference # The visited set pattern — every BFS/DFS uses this visited = set() visited.add(node) if node not in visited: ... # O(1) check
DSA connection: In BFS and DFS, the visited set is what prevents infinite loops. Using a list for visited is O(n) per check; a set makes it O(1). This is the difference between a working solution and a TLE.

Topic 5 of 8
Strings — methods, immutability & tricks

String fundamentals for DSA

strings are immutable arrays of chars
Strings behave like lists for indexing and slicing but are immutable — you cannot change a character in place. This means string concatenation in a loop is O(n²) — you must use join() instead.
Essential string methods
s = "Hello, World" # Case s.lower() # "hello, world" s.upper() # "HELLO, WORLD" # Search s.find("World") # 7 (index), -1 if not found s.startswith("He") # True s.endswith("ld") # True "World" in s # True # Split and join "a,b,c".split(",") # ['a', 'b', 'c'] ",".join(["a","b"]) # "a,b" " hello ".strip() # "hello" # Character checks "a".isalpha() # True "3".isdigit() # True "a3".isalnum() # True
The immutability trap & the correct join pattern
# SLOW — O(n²) — creates a new string every iteration result = "" for char in chars: result += char # bad in tight loops # FAST — O(n) — collect into list, join once result = "".join(chars) # the right way # String → list of chars (to "mutate" a string) s = "hello" chars = list(s) # ['h','e','l','l','o'] chars[0] = 'H' "".join(chars) # "Hello"
String tricks used constantly in interviews
# Palindrome check — two ways s[::-1] == s # simple reverse s == s[::-1] # Anagram check using sorted or Counter sorted("anagram") == sorted("nagaram") # True Counter("anagram") == Counter("nagaram") # True # ASCII value (for character math problems) ord('a') # 97 chr(97) # 'a' ord('c') - ord('a') # 2 ← used in trie problems # f-strings for debugging in interviews print(f"left={left}, right={right}, val={arr[left]}")
DSA connection: Sliding window on strings (longest substring without repeats, minimum window substring) uses Counter or defaultdict(int) to maintain a window frequency map — combine what you learned in topic 4 with string iteration.

Functions, scope & recursion basics

Functions — anatomy & scope

every DSA solution is a function
Function anatomy & argument types
# Basic def two_sum(nums: list, target: int) -> list: # type hints are optional but good practice seen = {} for i, n in enumerate(nums): if target - n in seen: return [seen[target-n], i] seen[n] = i return [] # Default arguments def binary_search(arr, target, lo=0, hi=None): if hi is None: hi = len(arr) - 1 # ... # Lambda — for sorting keys pairs = [(1,3), (2,1), (0,2)] pairs.sort(key=lambda p: p[1]) # sort by second element # [(2,1), (0,2), (1,3)]
Scope — the LEGB rule for DSA
# Problem: nested function can't reassign outer variable count = 0 def increment(): count += 1 # UnboundLocalError! # Fix 1: use nonlocal (for nested functions in recursion) def solve(): count = 0 def dfs(node): nonlocal count count += 1 # dfs traversal ... return count # Fix 2: use a mutable container (list/dict) def solve(): count = [0] # mutable — inner func can modify it def dfs(node): count[0] += 1 return count[0]

Recursion — the gateway to trees and graphs

DFS, backtracking, divide-and-conquer
Every recursive function has exactly two parts: a base case (when to stop) and a recursive call (a smaller version of the same problem). If you can't identify these two parts instantly, you don't understand the recursion yet.
Three progressively harder examples
# 1. Factorial — simplest recursion def factorial(n): if n <= 1: return 1 # ← base case return n * factorial(n-1) # ← recursive call # 2. Fibonacci with memoization def fib(n, memo={}): if n <= 1: return n if n in memo: return memo[n] # ← cache hit memo[n] = fib(n-1, memo) + fib(n-2, memo) return memo[n] # 3. Tree DFS — the exact pattern for tree problems def height(node): if node is None: return 0 # ← base case: null node left = height(node.left) # ← recurse left right = height(node.right) # ← recurse right return 1 + max(left, right)
Mental model for writing any recursion: (1) Trust that the function works for smaller inputs — don't trace through the call stack. (2) Write the base case(s) first. (3) Write the recursive case assuming the function already works. If you try to mentally trace a deep recursion, you'll confuse yourself every time.
Stack overflow warning: Python's default recursion limit is ~1000. For problems with depth > 1000 (deep graphs), use sys.setrecursionlimit(10000) at the start, or convert to iterative with an explicit stack.

Topic 7 of 8
Big-O complexity — thinking in time & space

Reading code for complexity

every interview answer needs this
Big-O measures how runtime grows as input size n grows. You are expected to state time AND space complexity at the end of every interview solution — unprompted.
Complexity from code patterns — learn to read this instantly
# O(1) — no loop, constant work return arr[-1] # O(log n) — input halved each step while lo <= hi: mid = (lo + hi) // 2 # binary search ... # O(n) — one loop over input for x in arr: ... # O(n log n) — sort, or loop + binary search inside arr.sort() # always O(n log n) # O(n²) — nested loops over same input for i in arr: for j in arr: # nested = multiply complexities ... # O(2ⁿ) — recursion where each call branches twice def fib(n): return fib(n-1) + fib(n-2) # exponential without memo
Space complexity — the other half
# O(1) space — only variables, no extra structures lo, hi = 0, len(arr)-1 # O(n) space — storing n elements seen = {} # up to n entries result = [x**2 for x in arr] # n elements # O(n) space — recursion call stack depth n def dfs(node): # call stack = tree height if not node: return dfs(node.left) dfs(node.right) # O(n*m) space — 2D DP table dp = [[0]*(m+1) for _ in range(n+1)]
The optimization move — hash map eliminates nested loop
# Two Sum — O(n²) naive for i in range(len(nums)): for j in range(i+1, len(nums)): if nums[i] + nums[j] == target: return [i, j] # Two Sum — O(n) optimized with hash map seen = {} for i, n in enumerate(nums): if target-n in seen: return [seen[target-n], i] seen[n] = i # O(n) time, O(n) space — classic tradeoff
Rule of thumb for interviews: n=10⁸ operations per second. O(n) with n=10⁶ → ~0.01s fine. O(n²) with n=10⁴ → 10⁸ ops → borderline. O(n²) with n=10⁶ → 10¹² ops → TLE (Time Limit Exceeded). Always ask: what's the constraint on n?

Putting it all together — 5 test problems

honest self-assessment
Before moving to actual DSA problems (arrays, two pointers, etc.), verify you can solve all 5 of these from scratch — no hints, no IDE. They test every prerequisite topic. If any trips you up, revisit that topic.
Problem 1 — lists + loops (should be easy)
# Given a list of numbers, return a new list # containing only the unique elements, preserving order. # Input: [3, 1, 4, 1, 5, 9, 2, 6, 5, 3] # Output: [3, 1, 4, 5, 9, 2, 6] # Answer — set for O(1) lookup, list to preserve order def unique_ordered(arr): seen = set() result = [] for x in arr: if x not in seen: result.append(x) seen.add(x) return result # Time: O(n) Space: O(n)
Problem 2 — dict + string
# Return the first non-repeating character in a string. # Input: "leetcode" Output: "l" # Input: "aabb" Output: "" def first_unique(s): count = Counter(s) for c in s: if count[c] == 1: return c return "" # Time: O(n) Space: O(1) — at most 26 entries
Problem 3 — two pointers + while loop
# Check if a string is a palindrome ignoring non-alphanumeric. # Input: "A man, a plan, a canal: Panama" Output: True def is_palindrome(s): s = s.lower() left, right = 0, len(s) - 1 while left < right: while left < right and not s[left].isalnum(): left += 1 while left < right and not s[right].isalnum(): right -= 1 if s[left] != s[right]: return False left += 1; right -= 1 return True # Time: O(n) Space: O(1)
Problem 4 — recursion
# Count the number of ways to climb n stairs # (1 or 2 steps at a time). This is exactly Fibonacci. # Input: n=4 Output: 5 def climb_stairs(n, memo={}): if n <= 2: return n if n in memo: return memo[n] memo[n] = climb_stairs(n-1, memo) + climb_stairs(n-2, memo) return memo[n] # Time: O(n) Space: O(n)
Problem 5 — complexity analysis
# State the time and space complexity of this function. def find_pairs(nums, target): result = [] seen = set() for n in nums: if target - n in seen: result.append((target-n, n)) seen.add(n) return result # Answer: # Time: O(n) — one loop, O(1) set operations # Space: O(n) — seen set can hold all n elements
